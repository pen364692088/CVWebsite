## 项目名称

**Ashen Archive**

## 项目目标

实现一个 **黑暗奇幻 + 极简现代** 风格的个人网站，面向 **Unity / 技术美术 / 游戏开发者**。

简历内容:"D:\成绩单\Resume\ZhouyuLiao-software-developer-resume.docx"
简历图片:"D:\成绩单\CV\300ppi"
作品视频:"D:\成绩单\CV\Work Collection of TA.mp4"

网站需要：

1. 视觉高级、有统一世界观气质
2. 运行流畅，不靠重型 3D 炫技
3. 支持桌面、平板、手机
4. 首页包含一个**轻量小游戏/互动模块**
5. 适合后续继续扩展作品项目与接单入口
6. 代码结构清晰，便于维护和二次开发

---

## 技术栈要求

### 必选框架

* **Next.js（App Router）**
* **React**
* **TypeScript**
* **Tailwind CSS**
* **Motion（原 Framer Motion）**

### 使用理由

* 用 **Next.js** 做页面结构、路由、元数据、图片优化、生产构建优化。([Next.js][1])
* 用 **React** 做组件化页面与模块拆分。([React][3])
* 用 **Tailwind CSS** 做响应式和深色主题风格控制。([Tailwind CSS][4])
* 用 **Motion** 做滚动、淡入、视差、悬停、分段 reveal 动画，不做重型动画引擎。([Motion][5])

### 禁止项

* 不要把主站做成全站 Three.js 首页
* 不要引入不必要的重型依赖
* 不要为了“酷”牺牲移动端性能
* 不要直接复刻黑魂 UI、图标、文案、具体视觉资产

## 设计方向

### 总体风格

**Dark Fantasy + Modern Minimal**

目标气质：

* 庄严
* 冷寂
* 残败
* 神秘
* 克制
* 不花哨
* 有探索感
* 有一点“仪式感”

### 设计原则

1. **深色为主，亮色稀缺**
2. **动效慢而重，不轻飘**
3. **排版现代，装饰克制**
4. **游戏感来自氛围和交互，不来自堆砌特效**
5. **网站首先是作品集，其次才是小游戏展示场**

## 配色方案

### 主色

* `#0B0D10` 近黑冷灰（主背景）
* `#171A1F` 深层背景 / 卡片背景
* `#6E737B` 雾灰 / 次级文字
* `#D8D3C8` 骨白 / 主标题文字
* `#8C7751` 暗金 / 边框 / hover 强调
* `#C96A2B` 火光橙 / 关键 CTA / 小游戏火种
* `#5A2A24` 深锈红 / 微弱危险感点缀

### 色彩规则

* 80% 深色背景
* 15% 中性灰 / 骨白
* 5% 暗金或火光橙点缀

## 字体方案

### 标题

选择 **庄重、雕刻感、古碑感 serif**
建议方向：

* Cinzel 风格
* Cormorant Garamond 风格
* Marcellus 风格

### 正文

选择 **现代、干净、可读性高 sans-serif**
建议方向：

* Inter
* Geist
* IBM Plex Sans
* Plus Jakarta Sans

### 字体原则

* 标题负责气质
* 正文负责清晰
* 不要全站花体
* 不要低对比细字

## 网站结构

### 页面结构

优先做 **单页主站 + 项目详情弹层/二级页**。

### 首页从上到下结构

1. **Hero / 入口场景**
2. **About / 角色档案**
3. **Core Disciplines / 能力区**
4. **Artifacts of Work / 项目区**
5. **Mini Game / 点燃篝火**
6. **Contact / Send a Signal**

---

## 每一屏具体要求

### 1. Hero

要有：

* 全屏深色背景
* 微弱雾气 / 余烬粒子 / 远处火光
* 主标题：`Ashen Archive`
* 副标题：`Unity Systems, Technical Art, VFX, and Playable Fragments`
* 两个按钮：

  * `Enter`
  * `View Projects`

交互要求：

* 鼠标轻微视差
* 滚动时首屏雾和火光有轻微变化
* 动画必须克制

### 2. About

形式像“角色档案”，不是普通简历块。
内容包括：

* 一句个人定位
* 你的核心方向
* 4 个重点能力标签
* 一句“可提供哪些帮助”的服务导向语句

### 3. Core Disciplines

用 4 张卡或 4 个符文式标签展示：

* Unity Development
* Technical Art
* VFX / Shader Work
* Tools & Optimization

要求：

* 每张卡 hover 有微弱发光或边框变化
* 不要做玻璃拟态
* 不要做赛博风

### 4. Artifacts of Work

项目区命名不要叫 Projects，改叫：
**Artifacts of Work**

每个项目卡要求：

* 项目封面图 / GIF
* 项目名
* 3 个标签
* 一句摘要
* 点击后展开详情

项目详情要求包含：

* 项目是什么
* 你做了什么
* 用了哪些技术
* 解决了什么问题
* 图片 / GIF / 视频

### 5. Mini Game

模块名：**Light the Fire**

玩法要求：

* 用户在黑暗中找到 3 个火种
* 收集火种后点燃篝火
* 点燃后解锁一个隐藏文案或额外 CTA

实现要求：

* 轻量
* 支持鼠标和触摸
* 可重玩
* 不依赖声音也能完整体验
* 不自动播放音频
* 不能喧宾夺主，时长控制在 20–45 秒

### 6. Contact

标题建议：

* `Send a Signal`
* 或 `Summon the Maker`

要求：

* 不是普通无聊表单页
* 但必须能快速联系到你
* 按钮：

  * Email
  * GitHub
  * LinkedIn
  * Resume

## 交互与动画要求

### 动画原则

* 少量高质，不要大量炫技
* 主要使用：

  * fade-in
  * slight upward reveal
  * parallax
  * slow glow
  * scroll-linked opacity / transform

### Motion 使用范围

* Hero 文案进场
* section reveal
* 项目卡 hover
* 小游戏状态切换
* contact 按钮的微反馈

### 禁止项

* 全屏粒子爆炸
* 高频闪烁
* 大面积 blur 导致性能下降
* 滚动时大量 JS 计算

---

## 响应式要求

### 目标设备

必须适配：

* 大屏桌面：1440+
* 笔记本：1280
* 平板：768–1024
* 手机：390 / 430 宽度

### 响应式规则

* Hero 在移动端不能信息挤爆
* 项目卡从 3 列降到 2 列再到 1 列
* 小游戏必须支持触摸操作
* Contact 区按钮在手机端易点击
* 字体层级在移动端要重新调，不允许机械缩放

---

## 性能要求

### 明确目标

* 首页首屏加载要快
* 移动端滚动不能明显掉帧
* 交互不能卡顿
* 动画尽量走 transform / opacity
* 图片必须优化
* 非关键资源延迟加载

### 实现要求

* 使用 `next/image`
* 控制首屏资源数量
* 所有图片提供明确尺寸
* 小游戏代码与首屏视觉解耦
* 项目详情支持懒加载
* 避免不必要的客户端大包
